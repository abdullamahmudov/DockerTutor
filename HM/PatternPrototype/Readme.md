# Инстркция к запуску  
Необходимо запускать методы тестов

# Структура  

1. PatternBase - проект с моделями данных  
2. TestsProj - проект с тестами  

## Описание моделей и реализаций методов клонирования  

### Основные модели данных  

1. `Food` - модель реализует метод поверхностного копирования интерфейса `IMyCloneable`  
2. `Furniture` - модель реализует метод глубокого копирования интерфейса `IMyCloneable`  
3. `Delivery` - модель реализует метод глубокого копирования интерфейса `ICloneable`  
4. `Installation` - модель реализует метод поверхностного копирования интерфейса `ICloneable`  
5. `Repair` - модель реализует метод поверхностного копирования интерфейса` ICloneable`  

## Реализованные тесты  

1. `SuperficialIMyCloneableTest` - Тестирование поверхностного клонирования интерфейса `IMyCloneable`
2. `DeepIMyCloneableTest` - Тестирование глубокого клонирования интерфейса `IMyCloneable`
3. `SuperficialICloneableTest` - Тестирование поверхностного клонирования интерфейса `ICloneable`
4. `DeepICloneableTest` - Тестирование глубокого клонирования интерфейса `ICloneable`

# Преимущества и недостатки  

## IClonable  
### Преимущества  
1. Готовый вариант интерфейса. Нет необходимости заводить лишний интерфейс в проекте.  

### Недостатки  
1. Из коробки интерфейс имеет лишь один метод для реализации и возможна реализовать лишь один вариант копирования (поверхностное, глубокое, пользовательский вариант).  
2. Тип возврашаемого значения у метода этого интерфейса `object` в связи с чем часто приходится использовать приведение типа.  

## IMyClonable  
### Преимущества  
1. Так как интерфейс реализуется пользователем есть, возможность сделать метод(ы) копирования такими какие необходимы в реализации того или иного решения.  

### Недостатки  
Не смог придумать явных. Возможно в случаях где недостаточно `ICloneable`, необходимо проектировать интерфес что затрчивает время.
