# DelegatesEvents

## Описание проекта

Проект включает следующие компоненты:

- **Мониторинг директории**: Автоматическое обнаружение новых файлов в папке `filesFolder`.
- **Генерация данных**: Создание случайных объектов с коллекциями элементов и запись их в JSON-файлы.
- **Обработка файлов**: Чтение файлов, десериализация объектов, поиск максимального элемента по весу и логирование результатов.
- **Логирование**: Запись всех действий в файл `logs/log.txt`.

Приложение использует многопоточность для фоновых задач, потокобезопасные операции с файлами и XML-документацию для методов.

### Архитектура
- **Interfaces/**: Интерфейсы для обработки информации и мониторинга директорий.
- **Models/**: Модели данных (FileEventArgs, ExampleClass, ExampleClassElement).
- **Implementations/**: Реализации компонентов (мониторы, логгеры, писатели, менеджеры процессов).
- **Extensions/**: Расширения для коллекций.

## Инструкция по запуску

1. **Требования**:
   - .NET 9.0 SDK (или выше).
   - ОС: Windows, macOS или Linux.

2. **Сборка проекта**:
   ```
   dotnet build
   ```

3. **Запуск приложения**:
   ```
   dotnet run --project ConsoleApp
   ```

4. **Использование**:
   - После запуска появится меню команд:
     1. Запустить прослушивание каталога файлов.
     2. Запустить запись случайных объектов в каталог.
     3. Остановить прослушивание каталога файлов.
     4. Остановить запись случайных файлов.
     5. Завершение программы.
   - Выберите команду, введя номер и нажав Enter.
   - Файлы будут создаваться в `bin/Debug/net9.0/filesFolder/`.
   - Логи — в `bin/Debug/net9.0/logs/log.txt`.

5. **Остановка**:
   - Выберите команду 5 для корректного завершения всех фоновых задач.
