# Как установить и включить OpenSSH?

## Цель

OpenSSH — это инструмент для безопасного удаленного входа с использованием протокола SSH. Он шифрует весь трафик между клиентом и сервером, предотвращая утечки данных и злонамеренные атаки.

ActiveProtect требует OpenSSH для резервного копирования баз данных SQL/Oracle и выполнения сценариев. Эта статья поможет вам установить и включить OpenSSH.

## Решение

### Windows

#### Перед началом работы

1. Запустите **Windows PowerShell** от имени администратора.
2. Выполните следующую команду, чтобы проверить, установлен ли сервер OpenSSH:
   ```powershell
   get-service sshd
   ```
3. На основе вывода:
   - Если sshd не указан, перейдите к **шагам установки**.
   - Если sshd установлен, но не запущен, следуйте **шагам включения**.

#### Установка сервера OpenSSH

Современные версии Windows включают OpenSSH как дополнительную функцию. Следуйте этим шагам для его установки:

1. Проверьте, доступен ли OpenSSH на вашей системе, выполнив:
   ```powershell
   Get-WindowsCapability -Online | Where-Object Name -like 'OpenSSH*'
   ```
2. На основе вывода:
   - Если OpenSSH указан, запомните точное имя службы и установите его, используя:
     ```powershell
     Add-WindowsCapability -Online -Name Exact Service Name
     ```
   - Если службы OpenSSH не отображаются, ваша версия Windows не включает OpenSSH. Загрузите файл MSI из [репозитория Windows на Github](https://github.com/PowerShell/Win32-OpenSSH/releases) и установите его, используя:
     ```powershell
     msiexec /i MSI File Path
     ```
   - Если появляется сообщение об ошибке, возможно, ваша виртуальная машина не подключена к Интернету. Обратитесь к [документации Windows](https://learn.microsoft.com/windows-server/administration/OpenSSH/openssh-overview), чтобы определить, можете ли вы установить его напрямую или нужно использовать GitHub.

#### Включение OpenSSH

1. После установки настройте OpenSSH на автоматический запуск:
   ```powershell
   Get-Service sshd | Set-Service -StartupType Automatic -PassThru
   Start-Service sshd
   ```
2. Чтобы проверить, работает ли OpenSSH, используйте:
   ```powershell
   get-service sshd
   ```

### Linux

Сервер OpenSSH встроен в большинство дистрибутивов Linux и может быть легко установлен с помощью простой команды:

- **Ubuntu/Debian**
  ```bash
  sudo apt-get install openssh-server
  ```

- **Arch**
  ```bash
  sudo pacman -S openssh
  ```

- **Red Hat**
  ```bash
  yum install  openssh-server
  ```

### Проверка порта OpenSSH

По умолчанию подключения SSH используют порт 22. Однако, если вы изменили настройки, SSH может использовать другой порт. Чтобы проверить, какой порт используется в данный момент, попробуйте один из следующих методов:

#### Метод 1: Проверьте файлы конфигурации

1. Откройте файл конфигурации SSH:
   - Windows: `%programdata%\ssh\sshd_config`
   - Linux: `/etc/ssh/sshd_config`
2. Найдите настройку **Порт** в файле. Если порт не указан, используется порт по умолчанию 22. Если есть несколько операторов порта, они указывают на ваши пользовательские настройки порта. Обратите внимание, что любая строка, начинающаяся с #, не действует.

#### Метод 2: Выполните команду

- **Windows**: Выполните следующую команду в PowerShell. Если установлен настраиваемый номер порта, он будет отображен в выводе. Если вывода нет, используется порт по умолчанию 22.
  ```powershell
  Select-String -Path "sshd_config" -Pattern "^Port" | ForEach-Object { $_.Line.Split()[1] }
  ```

- **Linux**: Выполните следующую команду в Терминале. Если установлен настраиваемый номер порта, он будет отображен в выводе. Если вывода нет, используется порт по умолчанию 22.
  ```bash
  grep -i '^port' /etc/ssh/sshd_config | awk '{print $2}'
  ```

#### Обновите номер порта в ActiveProtect

Если для SSH установлен нестандартный порт, вы должны ввести этот номер порта в ActiveProtect, чтобы обеспечить правильную работу заданий резервного копирования.

1. Выберите целевые виртуальные машины.
2. Нажмите **Ещё**, затем выберите **Указать адрес**.
3. Введите свой пользовательский номер порта.
